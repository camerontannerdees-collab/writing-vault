# Source Index

What lives in `source-material/`, organized so the agent can find things quickly. Update this when new source files are added — the agent can do it on demand.

## How source material is organized

Source material is anything you'd want to draw from but didn't write yourself: meeting notes, transcripts, articles, decks converted to markdown, research dumps, conversation captures. Every file lives in `source-material/` and gets cited by file path when drawn on.

Naming convention: `YYYY-MM-DD-short-description.md`. The date is the date of the source (when the meeting happened, when the article was published), not when it was added.

## How the agent uses this

When drafting, the agent reads relevant source files and cites them inline. When the source index doesn't list what's needed, the agent searches `source-material/` directly. If a needed source isn't there, the agent says so rather than making up a citation.
